建立專案

首先我會藉由 Rails 將一個專案給建立起來。

我想幫我的「歡樂書店」建立一個專案，會藉由 rails new 專案名稱：

為何後面還會看到 -j esbuild -c tailwind 呢？

-j esbuild

加入這個選項，指定了 JavaScript 要建構的工具，

這裡選擇esbuild，

esbuild 主要是一個快速將前端 JavaScript 代碼使用 esbuild 做打包以及編譯。

-c tailwind

加入這個選項，主要是定了你想要使用的 CSS 框架，

那這邊是指定Tailwind CSS，

Tailwind CSS 主要是一個工具包，可以快速建立，而減少有一些要撰寫自訂 CSS 的部分。

-j [esbuild | webpack | rollup]

只要有加任何參數 -j [esbuild | webpack | rollup]，

預設就會多出兩個檔案 Procfile.dev、bin/dev。

這樣就可藉由 bin/dev 取代以往的 rails server，將網站架起來。

藉由 http://localhost:3000/ 或是 http://127.0.0.1:3000/ 都可以查看 Rails 是否正常執行。

routes.rb 檔

首先，我們擁有一個網站，不外乎就是設置網站的路徑，

所謂的路徑，也可以想像有哪些網址可以通往哪裡？

我們藉由 routes.rb 檔，這個檔案主要就是放網站的路線可以通往哪裡。
